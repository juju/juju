# Architecture decision records

Architecture decision records (ADRs) document significant decisions that shape
Juju's internal architecture. Use ADRs when a decision affects subsystem
boundaries, durable data formats, controller and model behaviour, worker
coordination, or long-term maintenance.

New records should use numbered, slugged filenames:
`NNNN-short-decision-title.md`.
